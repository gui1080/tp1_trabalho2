Smoke Test

Trabalho 2 de TP1 por Guilherme Braga (17/0162290) e Gabriel Matheus (17/0103498)


1º Teste

Propósito:

Testar do ponto de vista de um novo usuário, ver a criação de um novo usuário.

Procedimento:
  1. Executar o Programa
  2. Criar conta
  3. Fechar o Programa

2º Teste

Propósito:

Testar do ponto de vista de um usuário recorrente, ver a validação de um usuário recorrente.

Procedimento:
  1. Executar o Programa
  2. Fazer login com sua conta que foi criada antes
  3. Fechar o Programa

3º Teste

Propósito:

Testar a segurança do usuário e ver sua validação.

Procedimento:
  1. Executar o Programa
  2. Fazer login com sua conta que não foi criada antes
  3. Ser impossibilitado de ingressar no sistema
  4. Fechar o Programa

4º Teste

Propósito:

Testar do ponto de vista de um usuário casual, testando a possibilidade de pesquisar possíveis Eventos.

Procedimento:
  1. Executar o Programa
  2. Selecionar a opção relacionada aos Eventos
  3. Checar os Eventos existentes, com suas Apresentações
  4. Fechar o Programa

5º Teste

Propósito:

Testar do ponto de vista do dono de um Evento, validar a criação de Evento.

Procedimento:
  1. Executar o Programa
  2. Fazer login
  3. Criar um novo Evento, informando todos os seus dados relacionados
  4. Fechar o Programa

6º Teste

Propósito:

Testar do ponto de vista do comprador, validar a venda de ingressos.

Procedimento:
  1. Executar o Programa
  2. Fazer login
  3. Escolher um Evento
  4. Comprar um ingresso
  5. Fechar o Programa

7º Teste

Propósito:

Testar do ponto de vista do comprador, validar a venda de ingressos.

Procedimento:
  1. Executar o Programa
  2. Fazer login
  3. Escolher um Evento que já está esgotado
  4. Tentar comprar um ingresso, e falhar
  5. Fechar o Programa

8º Teste

Propósito:

Testar do ponto de vista do comprador, retirar sua conta.

Procedimento:
  1. Executar o Programa
  2. Criar uma conta nova
  3. Entrar no sistema
  4. Efetuar descadastramento
  5. Terminar a operação com sucesso
  6. Fechar o Programa

9º Teste

Propósito:

Testar do ponto de vista do comprador.

Procedimento:
  1. Executar o Programa
  2. Entrar no sistema
  3. Ver os eventos disponíveis
  4. Comprar múltiplos ingressos de uma só vez
  5. Terminar a operação com sucesso
  6. Fechar o Programa

10º Teste

Propósito:

Testar do ponto de vista do comprador que talvez não tenha lido as especificações do evento com atenção (ou um possível erro de digitação).

Procedimento:
  1. Executar o Programa
  2. Entrar no sistema
  3. Ver os eventos disponíveis
  4. Tentar comprar milhões de ingressos de um evento com um número de ingressos limitado
  5. Falhar na operação
  6. Fechar o Programa

11º Teste

Propósito:

Testar do ponto de vista de quem organiza um determinado Evento, descadastrar um Evento.

Procedimento:
  1. Executar o Programa
  2. Entrar no sistema
  3. Criar um Evento novo
  4. Descadastrar o Evento logo em seguida
  5. Terminar a operação com sucesso
  6. Fechar o Programa

12º Teste

Propósito:

Evitar fraude.

Procedimento:
  1. Executar o Programa
  2. Entrar no sistema
  3. Criar um Evento novo para apenas 1 pessoa
  4. Sair do sistema
  5. Entrar novamente com outra conta
  6. Comprar o ingresso do Evento de 1 pessoa
  7. Sair do sistema
  8. Fazer login com a conta que é dona do Evento de teste
  9. Tentar apagar o Evento
  10. Falhar na operação
  11. Fechar o Programa

13º Teste

15º Teste

Propósito:

Testar premissões associadas a um usuário.
